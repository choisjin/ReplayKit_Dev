배포 빌드를 실행합니다. build_dist.py를 이용해 빌드 후 자동 배포합니다.

## 인자: $ARGUMENTS

인자가 없으면 전체 빌드 + 배포 (`--deploy`)를 실행합니다.
사용 가능한 인자: `--backend`, `--exe`, `--deploy-only`, `--clean`, `--init-deploy`

## 실행 절차

### 1. 사전 검사
- venv 활성화 상태 확인
- Cython, PyInstaller 설치 여부 확인
- 현재 git 상태 확인 (커밋되지 않은 변경사항 경고)

### 2. 빌드 실행
```bash
python build_dist.py --deploy
```
인자가 있으면 해당 인자로 실행:
```bash
python build_dist.py $ARGUMENTS
```

실행 시 live output을 표시하여 진행 상태를 확인합니다.

### 3. 결과 확인
- dist/ReplayKit/ 폴더의 파일 수, .pyd 수 확인
- 빌드 에러가 있으면 원인 분석 후 사용자에게 보고
- 배포 repo push 결과 확인

### 4. 완료 보고
빌드 결과를 요약하여 보고:
- 컴파일된 .pyd 파일 수
- frontend 빌드 성공 여부
- 배포 push 성공 여부
- 에러가 있었다면 에러 내용과 해결 방안

## 주의사항
- 빌드는 Windows 환경에서만 가능 (Visual Studio Build Tools 필요)
- frontend 빌드 전에 npm install이 완료되어 있어야 함
- 배포 repo가 초기화되지 않았으면 `--init-deploy`를 먼저 안내
- push 전에 사용자 확인을 받는다 (자동 push 금지)
