ReplayKit 사용자 가이드 테스트 자동화 플랫폼 — 녹화, 재생, 검증을 하나로

시스템 개요

ReplayKit은 Android 및 IVI(차량 인포테인먼트) 디바이스에서 사용자의 조작을 녹화하고, 이를 자동으로 재생하면서 스크린샷 기반 이미지 비교(SSIM)로 검증한 뒤 Excel 리포트를 생성하는 테스트 자동화 플랫폼입니다.

구성 요소기술역할
백엔드Python (FastAPI)디바이스 제어, 녹화/재생 엔진, 이미지 비교
프론트엔드React + TypeScript + Ant Design사용자 인터페이스 (웹 브라우저)
디바이스 연결ADB / TCP 소켓 / 시리얼Android, HKMC IVI, 시리얼 장치 등

설치 방법

방법 1: 설치 프로그램 사용 (권장)

  1. ReplayKit_Setup_1.0.exe 실행 → 관리자 권한 허용
  2. 설치 경로 확인 (기본: C:\ReplayKit)
  3. 구성 요소 선택:
    • ReplayKit Core — 기본 (필수, 내장 Python 포함)
    • DLT Viewer SDK — DLT 로그 뷰어 GUI를 사용하는 경우 체크 (Standard에 기본 포함)
    • Vimba X SDK — Vision Camera를 사용하는 경우 체크 (Full 선택 시 포함)
  4. 파일 복사가 완료되면 환경 설정 창(setup.bat)이 자동으로 열립니다
  5. Python은 내장(Embedded) 방식으로 포함되어 있어 별도 설치가 필요 없습니다
  6. Git이 없는 경우 — Git 설치 프로그램이 자동으로 실행됩니다:
    • 기본 설정 그대로 Next를 클릭하여 설치하세요
    • 설치 완료 후 아무 키를 눌러 진행합니다
  7. Python 패키지 자동 설치 → Git 저장소 초기화 → 완료
  8. 설치 완료 후 ReplayKit을 실행합니다
💡 내장 Python (Embedded Python) 배포판은 Python 3.10 Embedded를 포함하고 있어 시스템에 Python을 따로 설치할 필요가 없습니다. setup.bat 실행 시 자동으로 내장 Python에 pip 및 필요 패키지를 설치합니다. 개발 환경에서는 시스템 Python 3.10 + venv를 사용할 수도 있습니다.

방법 2: 수동 설치 (개발 환경)

# 프로젝트 폴더에서 초기 설정 (최초 1회)
setup.bat

# 내장 Python이 없으면 시스템 Python 3.10 + venv를 자동 생성합니다
# venv 생성, Python 패키지, Node 패키지를 모두 설치합니다

실행 방법

배포판 실행

ReplayKit.bat을 더블클릭하면 서버 관리 GUI가 실행됩니다.

개발 환경: 서버 관리 GUI

python server.py

Tkinter 기반 GUI 창이 열리며, 백엔드와 프론트엔드를 동시에 관리할 수 있습니다. 서버가 준비되면 브라우저가 자동으로 열립니다.

개발 환경: 수동 실행

# 터미널 1: 백엔드
venv\Scripts\activate
python -m uvicorn backend.app.main:app --reload --port 8000

# 터미널 2: 프론트엔드
cd frontend
npm run dev

실행 후 브라우저에서 http://localhost:5173에 접속하면 ReplayKit 화면이 표시됩니다.

💡 팁 프론트엔드가 백엔드의 API 요청을 자동으로 프록시하므로, 별도의 CORS 설정이 필요 없습니다.

업데이트

ReplayKit은 두 가지 방법으로 업데이트할 수 있습니다:

방법 1: 웹에서 업데이트 (권장)

  1. 좌측 메뉴의 업데이트 버튼 클릭
  2. 확인 대화상자에서 확인 클릭
  3. 화면에 "서버 업데이트 중..." 대기 화면이 표시됩니다
  4. 서버가 자동으로 종료 → git pull → 재시작됩니다
  5. 서버가 준비되면 자동으로 새로고침됩니다

방법 2: ReplayKit.bat 재실행

프로그램을 완전히 종료하고 ReplayKit.bat을 다시 실행하면 시작 시 자동으로 git pull이 실행되어 최신 상태로 업데이트됩니다.

💡 업데이트 사용자 데이터 보존 시나리오, 테스트 결과, 디바이스 설정, 경로 설정 등 사용자 데이터는 git 추적 대상이 아니므로 업데이트 시 변경되지 않습니다.

런처

ReplayKit 런처는 최소화된 컨트롤 패널로, 서버 관리 기능을 제공합니다.

버튼기능
▶ / ■서버 시작/정지 토글. 실행 중이면 빨간 ■, 정지 상태면 녹색 ▶
↻서버 재시작 (git pull + 서버 재시작)
🌐웹 브라우저 열기
━시스템 트레이로 최소화
✕프로그램 완전 종료
💡 시스템 트레이 런처 창을 닫으면(━ 버튼 또는 X 버튼) 시스템 트레이로 최소화됩니다. 트레이 아이콘을 우클릭하면 서버 시작/정지/재시작/웹 열기/종료 메뉴가 표시됩니다. 트레이 아이콘을 더블클릭하면 런처 창이 다시 나타납니다.
💡 런처 로그 확인 웹 화면 좌측 상단의 ReplayKit 로고를 5번 연속 클릭하면 런처 로그 창이 나타나 서버 시작/동기화/업데이트 과정의 로그를 확인할 수 있습니다.

디바이스 관리

ReplayKit의 첫 화면인 디바이스 탭에서 테스트에 사용할 장치들을 등록하고 관리합니다. 디바이스는 주 디바이스와 보조 디바이스로 나뉘며, 같은 타입의 디바이스끼리 그룹으로 묶여 표시됩니다.

💡 Device ID 규칙 각 디바이스에는 모델명_번호 형식의 고유 ID가 부여됩니다. 예: Connected_Wide_1, CAN_2. 그룹 내에서 드래그앤드롭으로 순서를 변경하면 번호가 자동 재할당됩니다.

주 디바이스

화면 조작과 스크린샷 캡처의 대상이 되는 디바이스입니다.

유형연결 방식설명
ADB USB 또는 WiFi Android 디바이스. USB로 연결하거나 IP:포트로 WiFi 연결.
예: 192.168.1.100:5555
HKMC TCP 소켓 HKMC 6세대 IVI 시스템. IP와 TCP 포트(기본 5000)로 연결.
화면 유형 선택 가능: Front Center, Rear Left, Rear Right, Cluster
Vision Camera GigE Vision (UDP) GigE Vision 산업용 카메라. 스캔 시 GVCP 프로토콜로 자동 감지되며, MAC 주소, 현재 IP, 서브넷 정보가 표시됩니다. Vimba X SDK 설치 시 추가 기능 사용 가능.

디바이스 연결하기

  1. 주 디바이스 추가 또는 보조 디바이스 추가 버튼 클릭
  2. 주 디바이스의 경우 프로젝트와 장비 모델을 선택합니다. 모델에 따라 Device ID prefix가 결정됩니다 (예: GVM → GVM_1)
  3. 스캔으로 자동 감지된 기기를 선택하거나, 수동 연결 탭에서 주소를 직접 입력합니다
  4. 보조 디바이스의 경우 모듈을 선택하면 모듈명이 Device ID prefix가 됩니다 (예: CAN → CAN_1)
⚠️ ADB 디바이스 준비 Android 기기의 설정 → 개발자 옵션 → USB 디버깅이 켜져 있어야 합니다. WiFi 연결 시에는 adb tcpip 5555 명령이 먼저 실행되어 있어야 합니다.

Vision Camera 연결 (GigE Vision)

스캔 시 네트워크에 연결된 GigE Vision 카메라가 자동으로 감지됩니다. 감지 결과에 카메라의 현재 IP와 PC 네트워크 인터페이스 IP가 함께 표시됩니다.

  1. 스캔에서 카메라가 감지되면 MAC, 모델, 현재 IP가 표시됩니다
  2. 카메라 IP가 PC와 다른 서브넷에 있다면 IP 변경 버튼을 클릭합니다
  3. ForceIP 대화상자에서 PC 서브넷에 맞는 IP가 자동으로 추천됩니다
  4. 확인을 누르면 카메라 IP가 변경되고, 자동으로 다시 스캔됩니다
  5. 이제 연결 버튼을 클릭하면 수동 연결 탭에 값이 채워지며, 연결 버튼으로 완료합니다
💡 ForceIP란? GigE Vision 표준의 IP 강제 설정 기능입니다. 카메라가 PC와 다른 서브넷에 있어 통신이 불가능할 때, 카메라의 IP를 PC와 같은 서브넷으로 변경하여 연결할 수 있게 합니다. 이 설정은 카메라 재부팅 시 초기화될 수 있습니다.

보조 디바이스

주 디바이스를 보조하는 외부 장치들입니다. 시리얼 통신, 모듈 제어 등에 사용됩니다.

유형연결 방식설명
시리얼 COM 포트 / USB 시리얼 통신 장치. 보율(Baudrate) 설정 가능 (기본 115200)
모듈 플러그인 확장 모듈 (CAN 통신, 전원 제어 등). 동적으로 로딩되는 플러그인
벤치 UDP 벤치 장비. IP:UDP 포트로 연결

디바이스 작업

연결 상태 표시

각 디바이스는 현재 연결 상태를 실시간으로 표시합니다:

상태색상설명
연결됨초록디바이스가 정상적으로 연결되어 사용 가능
연결 중파랑(깜빡임)연결을 시도하고 있는 상태
연결 끊김회색등록은 되어 있지만 연결되지 않은 상태
오프라인빨강디바이스를 찾을 수 없음
오류빨강연결 시도 중 오류 발생

그룹 및 순서 관리

디바이스는 같은 타입(Device ID prefix)끼리 그룹 카드로 묶여 표시됩니다. 그룹 내에서 드래그앤드롭(좌측 ⠿ 핸들)으로 순서를 변경하면 Device ID 번호가 자동으로 재할당됩니다.

디바이스를 삭제하면 같은 그룹의 남은 디바이스 번호가 자동으로 1부터 재정렬됩니다.

디바이스 정보 수정

편집 버튼을 클릭하면 수정 모달이 열립니다:

  • Device ID (prefix) — 드롭다운에서 변경 가능. 주 디바이스는 프로젝트별 모델 목록, 보조 디바이스는 모듈 목록이 표시됩니다. 변경 시 새 그룹의 마지막 번호 다음으로 자동 배정됩니다.
  • 이름 — 디바이스 표시명 변경
  • Baudrate — 시리얼 디바이스의 통신 속도 변경
  • 모듈 — 보조 디바이스의 연결 모듈 변경

개별 디바이스 버튼

각 디바이스 항목 우측에 상태에 따라 버튼이 표시됩니다:

  • 연결 — 연결 끊김 상태의 디바이스에 표시. 클릭하면 연결을 시도합니다
  • 연결 끊기 — 연결된 디바이스에 표시. 클릭하면 연결만 끊고 등록은 유지합니다
  • 편집 — 디바이스의 이름, 주소, 설정 등을 수정합니다
  • 삭제 — 디바이스 등록을 완전히 해제합니다 (연결도 끊김, 번호 자동 재정렬)

일괄 작업 버튼

디바이스 목록 상단에 일괄 작업 버튼이 있습니다:

  • 전체 연결 — 등록된 모든 디바이스에 연결을 시도합니다
  • 선택 연결 (N) — 체크박스로 선택한 디바이스만 연결합니다
  • 전체 연결 끊기 — 모든 디바이스의 연결을 끊습니다
  • 선택 연결 끊기 (N) — 선택한 디바이스만 연결을 끊습니다
💡 서버 시작 시 기본 상태 서버가 시작되면 모든 디바이스는 연결 끊김 상태입니다. 사용자가 디바이스 탭에서 전체 연결 또는 개별 연결 버튼을 클릭하여 수동으로 연결해야 합니다. 연결을 끊으면 자동 재연결이 비활성화됩니다. 다시 연결 버튼을 누르면 자동 재연결이 복구됩니다.

녹화

녹화 탭에서는 디바이스의 실시간 화면을 보면서 사용자 조작을 기록하여 테스트 시나리오를 만듭니다. 각 조작은 하나의 스텝(Step)으로 저장됩니다.

녹화 시작하기

비녹화 상태 (녹화 제어 카드)

  1. 녹화 제어 카드의 시나리오 콤보에서 기존 시나리오를 선택하거나, 새로 만들기 버튼으로 새 시나리오를 생성합니다
  2. 새로 만들기 클릭 시 모달에서 시나리오 이름을 입력합니다. 중복 이름일 경우 덮어쓰기 또는 이름 변경을 선택할 수 있습니다
  3. 필요시 설명을 입력합니다
  4. 녹화 시작 버튼을 클릭합니다

시나리오 관리 버튼

시나리오 콤보 옆에 관리 버튼이 있습니다 (시나리오가 선택된 상태에서):

  • 복사 — 현재 시나리오를 새 이름으로 복사합니다
  • 이름 변경 — 시나리오 이름을 변경합니다
  • 삭제 — 시나리오를 삭제합니다 (확인 필요)

녹화 중 상태

녹화가 시작되면 화면 레이아웃이 변경됩니다:

  • 상단 바: 시나리오 이름(읽기 전용) + 설명 입력 + 우측에 ● 녹화 중 태그와 녹화 중지 버튼
  • 스텝 수동 추가 카드: 1행에 대상 디바이스 + 스텝 타입, 2행에 파라미터 입력 + delay. 우측 상단에 + 스텝 추가 버튼
  • 기록된 스텝 목록: 녹화된 스텝이 실시간으로 추가됩니다

디바이스 화면을 클릭/드래그하면 자동으로 스텝이 기록됩니다. 녹화 중에도 스텝 순서 변경, Wait 삽입, 기대 이미지 캡처 등 모든 편집 기능을 사용할 수 있습니다.

⚠️ 저장 확인 녹화 중지 후 다른 페이지로 이동하거나 브라우저를 닫으면 저장 여부를 확인하는 대화상자가 나타납니다. 스텝이 변경된 상태에서 저장하지 않고 이동하면 변경 사항이 사라집니다.
💡 이어서 녹화 이미 저장된 시나리오를 콤보에서 선택하면 해당 시나리오의 스텝이 로드됩니다. 이어서 녹화 버튼으로 기존 스텝 뒤에 새 스텝을 추가할 수 있습니다.

수동 스텝 추가

녹화 중이거나 편집 모드에서 화면 하단의 스텝 추가 영역을 사용합니다. 스텝 유형을 선택하고 필요한 값을 입력한 뒤 추가 버튼을 클릭합니다.

기대 이미지

각 스텝이 올바르게 실행되었는지 확인하려면 기대 이미지를 설정합니다. 재생 시 실제 스크린샷과 기대 이미지를 비교하여 Pass/Fail을 판정합니다.

기대 이미지 캡처하기

  1. 스텝 목록 우측의 📷 카메라 아이콘을 클릭합니다
  2. 비교 모드 선택 팝업이 나타납니다:
    • 전체화면 — 현재 화면을 즉시 캡처
    • 단일크롭 — 크롭 편집 모달이 열림
    • 영역제외 — 제외 영역 편집 모달이 열림
    • 멀티크롭 — 복수 크롭 편집 모달이 열림
  3. 캡처 완료 후 스텝 2행 우측에 📷 FULL, 📷 CROP 등의 태그가 표시됩니다
  4. 태그를 클릭하면 기대 이미지를 미리볼 수 있습니다 (영역제외/멀티크롭 시 박스 오버레이 포함)
  5. 태그 옆의 ✕ 버튼으로 기대 이미지를 초기화할 수 있습니다
💡 비교 모드 변경 시 주의 카메라 아이콘으로 새 비교 모드를 선택하면 이전 모드의 데이터(크롭 영역, 제외 영역 등)가 자동으로 초기화됩니다. 예: CROP → FULL로 변경하면 기존 ROI 정보가 삭제됩니다.

이미지 비교 모드

기대 이미지와 실제 스크린샷을 비교하는 4가지 모드를 제공합니다:

모드아이콘설명사용 상황
전체화면 FULL 화면 전체를 SSIM으로 비교 화면 전체가 동일해야 할 때 (기본값)
단일크롭 SINGLE_CROP 특정 영역만 잘라서 템플릿 매칭 상태바, 알림 등 특정 UI 요소만 확인할 때
영역제외 FULL_EXCLUDE 지정 영역을 제외하고 SSIM 비교 시계, 배터리 등 변하는 영역을 무시할 때
멀티크롭 MULTI_CROP 여러 영역을 각각 비교 (모두 통과해야 Pass) 여러 UI 요소를 동시에 확인할 때

비교 영역 편집

  • 영역제외 편집 (가위 아이콘) — 기대 이미지 위에서 마우스로 드래그하여 제외할 영역(빨간색)을 지정
  • 크롭 영역 편집 — 비교할 영역(초록색)을 드래그하여 지정
  • ROI 설정 — 전체화면 모드에서 비교 대상 영역만 지정
  • 각 영역은 목록에서 클릭하여 수정하거나 X 버튼으로 삭제할 수 있습니다
💡 유사도 임계값 기본 유사도 임계값은 0.95 (95%)입니다. 이 값 이상이면 Pass, 미만이면 Fail로 판정됩니다. 멀티크롭 모드에서는 각 크롭별로 개별 임계값을 설정할 수 있습니다.

스텝 편집

기록된 스텝은 언제든 수정할 수 있습니다. 각 스텝에는 두 줄의 정보가 표시됩니다:

1행: 핵심 정보

  • 인덱스 번호 — 스텝 순서 (1부터 시작)
  • 설명 (Remark) — 스텝에 대한 설명. 직접 입력 가능
  • 함수/파라미터 — 스텝의 실행 내용 (예: tap (70,69), {"x":65,"y":169})
  • Delay (ms) — 스텝 실행 후 대기 시간 (우측 정렬). 직접 수정 가능

2행: 부가 정보

  • 디바이스 태그 — 이 스텝이 실행될 디바이스
  • 타입 태그 — tap, wait, module_command 등
  • 점프 태그 — 조건부 이동이 설정된 경우 P→#N, F→END 등으로 표시
  • 기대 이미지 태그 — 📷 FULL / CROP / EXCLUDE / MULTI (우측 정렬). 클릭 시 미리보기

우측 아이콘 영역 (2행)

↑ ↓스텝 순서 변경 (위/아래 이동)
⚡이 스텝만 단독 실행 + 검증 (디바이스 미연결 시 비활성화)
🗑스텝 삭제 (확인 필요)
2행기능
✏️커맨드 수정 — 좌표, 텍스트, 명령어 등 파라미터 변경
🔀조건부 이동 설정 — Pass/Fail 시 점프할 스텝 지정
WWait 삽입 — 이 스텝 뒤에 대기 스텝 추가 (모드 선택 팝업)
📷기대 이미지 캡처 — 비교 모드 선택 팝업

Device 일괄 전환

스텝 카드 상단의 ⇄ Device 전환 버튼을 클릭하면, 시나리오에 사용된 모든 Device ID 목록이 표시됩니다. 각 ID 옆의 드롭다운에서 변경할 디바이스를 선택하고 적용을 클릭하면 모든 스텝의 device_id와 screenshot_device_id가 일괄 변경됩니다.

💡 활용 예시 다른 PC에서 만든 시나리오를 가져왔을 때, 또는 디바이스를 교체했을 때 Device ID를 일괄 변경하여 바로 재생할 수 있습니다.

하드키 롱프레스

HKMC 디바이스의 하드키 버튼을 길게 누르면 빨간색 게이지가 차오릅니다. 3초 이상 유지 시 버튼 전체가 빨간색으로 변하며 롱프레스(Long)로 인식됩니다. 게이지가 차기 전에 놓으면 숏프레스(Short)로 처리됩니다.

화면 크기 자동 조정

세로형 디바이스(스마트폰 등)에서 화면이 카드를 가득 채워 슬라이더가 보이지 않는 문제를 방지합니다. 화면 영역 자르기 또는 하드키 패널이 활성화되면 화면 영역이 자동으로 축소되어 하단에 컨트롤 공간이 확보됩니다. 좌표 변환은 화면 비율 기반으로 계산되므로 정확도에 영향 없습니다.

💡 줄무늬 배경 스텝 목록은 홀짝 줄에 번갈아 배경색이 적용되어 스텝 간 구분이 쉽습니다.

Wait 스텝 (대기)

Wait 스텝은 다음 스텝 실행 전에 지정된 시간만큼 대기합니다. W 버튼 또는 Wait 추가 버튼을 클릭하면 3가지 모드를 선택할 수 있습니다:

모드설명파라미터사용 예
기본 지정된 시간(ms)만큼 고정 대기 대기 시간 (ms) 화면 전환 후 1초 대기: 1000ms
사이클 반복 재생 시 회차마다 대기 시간이 증가 시작(ms) + 간격(ms) 시작 3000ms, 간격 3000ms, 3회 반복 시:
1회차: 3000ms, 2회차: 6000ms, 3회차: 9000ms
랜덤 지정 구간 내에서 랜덤 시간 대기 최소(ms) ~ 최대(ms) 0~10000ms 범위: 매번 실행할 때마다 다른 시간 대기
💡 사이클 모드 활용 반복 테스트에서 점진적으로 대기 시간을 늘려야 할 때 유용합니다. 예: 서버 부하 테스트에서 요청 간격을 점점 늘리는 경우.

조건부 이동

스텝의 검증 결과(Pass/Fail)에 따라 다음 실행할 스텝을 지정할 수 있습니다. 이를 통해 테스트 흐름을 분기하거나, 실패 시 건너뛰기 등의 로직을 구현합니다.

설정설명
Pass 시 이동검증 성공 시 지정한 스텝으로 점프 (기본: 다음 스텝)
Fail 시 이동검증 실패 시 지정한 스텝으로 점프 또는 종료(END)
⚠️ 무한 루프 주의 조건부 이동을 설정할 때, 이전 스텝으로 돌아가는 순환 참조가 발생하지 않도록 주의하세요.

시나리오 관리

시나리오 탭은 좌우 2패널 구조입니다. 좌측에 시나리오 목록, 우측에 시나리오 상세/재생 결과가 표시됩니다. 패널 크기는 가운데 구분선을 드래그하여 조절할 수 있습니다.

상단 도구 버튼 (우측 정렬)

  • 그룹 관리 — 시나리오 그룹 생성/편집
  • 시나리오 합치기 — 여러 시나리오를 하나로 병합
  • 시나리오 내보내기 — ZIP 파일로 패키징
  • 시나리오 가져오기 — ZIP 파일에서 불러오기
  • 새로고침 — 목록 갱신

시나리오 목록 (좌측 패널)

저장된 모든 시나리오가 목록으로 표시됩니다. 그룹별 탭과 전체 탭으로 구분하여 볼 수 있습니다.

시나리오를 클릭하면 우측 패널에 스텝 상세가 표시됩니다.

시나리오 상세 (우측 패널)

선택한 시나리오의 스텝 목록이 테이블로 표시됩니다. 타이틀 영역에 관리 및 재생 버튼이 있습니다:

  • 이름 변경 — 시나리오 이름 수정
  • 복사 — 새 이름으로 복사
  • 반복 횟수 — 재생 반복 횟수 설정 (기본 1회)
  • 재생 — 시나리오 실행 (디바이스 미연결 시 오류 메시지 표시)
  • 웹캠 녹화 — 재생 중 웹캠으로 화면 녹화
  • 삭제 — 시나리오 및 관련 스크린샷을 함께 제거
  • 그룹 추가 — 시나리오를 특정 그룹에 배정

스텝 테이블 컬럼

컬럼설명
체크박스재생 시 건너뛸 스텝을 선택/해제 (체크 해제 = 건너뜀)
#스텝 인덱스 (1부터)
Type스텝 유형 (tap, wait, module_command 등)
Device대상 디바이스
설명스텝 설명
Delay대기 시간 (ms). 직접 수정 가능
비교기대 이미지 미리보기 (설정된 경우)

재생 (실행)

시나리오를 실행하여 자동으로 디바이스를 조작하고 결과를 검증합니다.

⚠️ 디바이스 연결 필수 재생 전에 시나리오에서 사용하는 모든 디바이스가 연결됨 상태여야 합니다. 연결되지 않은 디바이스가 있으면 오류 메시지가 표시되고 재생이 시작되지 않습니다. 디바이스 탭에서 먼저 연결하세요.

Preflight 검사 및 디바이스 매핑

재생 시작 전에 시나리오에 사용된 디바이스가 현재 연결된 디바이스와 일치하는지 preflight 검사가 자동 수행됩니다. 디바이스 ID가 다르거나 연결이 안 된 경우 디바이스 매핑 변경 대화상자가 표시됩니다:

  • 시나리오에 기록된 디바이스와 현재 연결된 디바이스를 대조하여 표시
  • 불일치하는 디바이스를 현재 연결된 다른 디바이스로 재매핑 가능
  • 매핑을 변경하면 해당 재생에만 적용되며 원본 시나리오는 수정되지 않습니다
  1. 시나리오 상세 타이틀의 재생 버튼을 클릭합니다
  2. 디바이스 연결 상태가 자동으로 확인됩니다
  3. 필요 시 반복 횟수를 설정합니다 (기본 1회)
  4. 각 스텝이 실행될 때마다 결과가 실시간으로 표시됩니다:
    • Pass — 유사도가 임계값 이상
    • Warning — 경계값 수준
    • Fail — 유사도가 임계값 미만
    • Error — 실행 오류 발생

재생 전 절전 모드 확인

재생 시작 시 PC의 절전 모드 설정을 자동으로 확인합니다. 절전 모드가 활성화되어 있으면 경고 대화상자가 표시됩니다:

  • 절전 차단 후 재생 — 재생 중 절전 모드를 임시로 차단합니다. 재생 완료 후 자동 복원됩니다.
  • 취소 — 재생을 취소합니다. Windows 설정에서 직접 절전을 해제한 뒤 다시 시도할 수 있습니다.
⚠️ 장시간 재생 시 주의 PC가 절전 모드에 들어가면 네트워크 연결이 끊겨 재생이 비정상 중단될 수 있습니다. 반복 재생 등 장시간 작업 시에는 절전 차단 옵션을 사용하세요.

반복 재생 시 중간 결과 저장

반복 횟수를 2회 이상으로 설정한 경우, 각 사이클이 완료될 때마다 결과가 자동 저장됩니다. 비정상 종료(절전, 네트워크 끊김 등) 시에도 완료된 사이클까지의 결과가 보존되어 테스트 결과 페이지에서 확인할 수 있습니다.

💡 WebSocket 끊김 시 자동 중지 브라우저와 서버 간의 WebSocket 연결이 끊기면 재생이 자동으로 중지됩니다. 완료된 사이클까지의 결과는 저장됩니다.

그룹 관리

여러 시나리오를 그룹으로 묶어 순차적으로 실행할 수 있습니다. 상단의 그룹 관리 버튼으로 그룹을 생성하고 관리합니다.

  1. 그룹 만들기 — 그룹 이름을 입력하고 생성
  2. 그룹에 시나리오를 추가 (여러 개 가능)
  3. 드래그로 시나리오 순서 변경
  4. 필요 시 시나리오 간 조건부 이동 설정:
    • 시나리오 A의 Pass → 시나리오 C로 점프
    • 시나리오 A의 Fail → 종료(END)
  5. 그룹 재생을 클릭하면 포함된 시나리오가 순차적으로 실행됩니다

그룹 재생 결과 저장 구조

그룹 재생 시 각 시나리오의 스크린샷은 results/{결과폴더}/{인덱스}_{시나리오명}/screenshots/ 구조로 저장됩니다. 인덱스는 그룹 내 실행 순서(0부터 시작)를 나타냅니다.

내보내기 / 가져오기

내보내기 (Export)

시나리오와 그룹을 ZIP 파일로 패키징하여 다른 PC에 배포할 수 있습니다.

  1. 상단 시나리오 내보내기 버튼 클릭
  2. 내보낼 시나리오/그룹을 선택
  3. 내보내기 클릭 → ZIP 파일이 생성됩니다

가져오기 (Import)

  1. 상단 시나리오 가져오기 버튼 클릭 → ZIP 파일 선택
  2. 미리보기 화면에서 포함된 시나리오 목록과 충돌 여부를 확인합니다
  3. 충돌 시 처리 방식을 선택합니다:
    • 건너뛰기 — 기존 시나리오 유지
    • 덮어쓰기 — 가져오는 시나리오로 교체
    • 이름 변경 — 새 이름으로 추가
  4. 적용을 클릭하면 가져오기가 완료됩니다

테스트 결과

테스트 결과 탭에서는 시나리오 재생 결과를 확인하고 리포트를 생성합니다.

결과 목록

모든 재생 결과가 최신순으로 나열됩니다. 각 항목에는 다음 정보가 표시됩니다:

  • 시나리오 이름
  • 실행 일시 (24시간 형식, 날짜와 시간이 두 줄로 표시)
  • 전체 스텝 수 / Pass / Fail 수
  • 상태 (전체 결과)

시나리오 이름으로 필터링하거나, 선택 후 일괄 삭제도 가능합니다.

스크린샷 저장 구조

재생 시 생성되는 스크린샷은 results/{결과폴더}/screenshots/ 하위에 직접 저장됩니다. 기대 이미지, 실제 캡처, 차이점(diff) 이미지가 각 스텝별로 관리됩니다.

레거시 결과 정리

이전 버전에서 생성된 플랫 파일 구조의 결과가 남아 있는 경우, 레거시 정리 버튼이 표시됩니다. 클릭하면 오래된 결과 파일을 정리하고 새 구조로 마이그레이션합니다.

결과 상세

결과를 클릭하면 각 스텝별 상세 정보를 볼 수 있습니다. 상단의 필터를 사용하여 원하는 결과만 표시할 수 있습니다:

  • Status — Pass / Warning / Fail / Error 별 필터링
  • Device — 특정 디바이스의 스텝만 표시
  • Repeat — 반복 재생 시 특정 회차만 표시

Excel 내보내기

결과를 Excel(.xlsx) 리포트로 내보낼 수 있습니다. 리포트에는 다음이 포함됩니다:

  • 시나리오 요약 정보
  • 스텝별 실행 결과 표
  • 기대/실제/차이점 스크린샷 이미지 (Excel에 삽입)
  • 상태별 색상 구분

설정에서 Excel 저장 경로를 지정하면 해당 경로에 자동 저장되며, 지정하지 않으면 브라우저 다운로드로 저장됩니다.

팁 & 문제 해결

자주 묻는 질문

Q: 디바이스가 스캔에 나타나지 않아요

확인 사항:
  • Android: USB 디버깅이 켜져 있는지 확인
  • Android: USB 케이블을 다시 연결하고 PC에서 허용 팝업이 뜨면 승인
  • HKMC: 동일 네트워크에 있는지, 방화벽이 차단하지 않는지 확인
  • 시리얼: 드라이버가 설치되어 있는지, 다른 프로그램이 COM 포트를 점유하고 있지 않은지 확인
  • ADB 서버 재시작을 시도해보세요

Q: 재생 시 유사도가 낮게 나와요

  • 시간에 따라 변하는 영역(시계, 배터리 등)이 있다면 영역제외 모드를 사용하세요
  • 화면 전체가 아닌 특정 요소만 확인하고 싶다면 단일크롭 또는 멀티크롭을 사용하세요
  • 유사도 임계값을 조정할 수 있습니다 (기본 0.95)
  • 디바이스 해상도가 녹화 시와 다르면 차이가 발생할 수 있습니다

Q: 다른 PC에서 시나리오를 실행하고 싶어요

  • 내보내기로 시나리오를 ZIP 파일로 패키징
  • 다른 PC에서 가져오기로 ZIP 파일을 불러옵니다
  • 재생 시 디바이스 매핑 단계에서 현재 PC의 디바이스를 선택하면 됩니다

Q: 서버 연결이 끊어졌어요

  • 화면 상단에 "서버 연결 중..." 메시지가 표시됩니다
  • 자동 재연결이 시도되며, 서버가 다시 실행되면 자동으로 복구됩니다
  • 해결되지 않으면 server.py를 다시 실행하세요

Q: 스텝 실행 중간에 대기 시간을 조절하고 싶어요

각 스텝의 커맨드 수정에서 delay_after_ms 값을 변경합니다. 기본값은 1000ms (1초)이며, 화면 전환이 느린 경우 늘리면 됩니다.

Q: "서버가 이미 실행 중입니다" 메시지가 나와요

ReplayKit은 서버 중복 실행을 자동으로 감지합니다. 이미 실행 중인 인스턴스가 있으면 대화상자가 표시되며, 기존 프로세스를 종료 후 다시 시작할 수 있습니다.

서버 자동 관리 기능

  • 고아 결과 폴더 자동 정리 — 서버 시작 시 result.json 파일이 없는 불완전한 결과 폴더를 자동으로 삭제합니다.
  • 로그 날짜별 저장 — 서버 로그는 날짜별 파일로 관리되며, 7일이 지난 로그는 자동으로 삭제됩니다.
